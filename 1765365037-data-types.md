# Data Types

## CHAR(n)

`CHAR(n)` es un string con una logintud concreta de caracteres dados por `n`.
Si se necesita alguna codificación de algún idioma esta codificación se pone de esta manera => 
`CHARACTER SET ascii` ascii se puede sustituir por cualquier otra codificación com UTF8.

Ejemplos de como se usuaria char =>

```sql
nombre_atributo CHAR(4)

```
Especificando la codificación =>

```mysql
nombre_atributo CHAR(6) CHARACTER SET ascii
```


## DATE, DATETIME, TIMESTAMP, YEAR and TIME

### DATE

- Formato -> `'YYYY-MM-DD'`
- Rango   -> `'1000-01-01'` to `'9999-12-31'`

### DATETIME

- Formato -> `'YYYY-MM-DD HH:MM:SS'`
- Rango   -> `'1000-01-01 00:00:00'` to `'9999-12-31 23:59:59'`

### TIMESTAMP

==ES DE TIPO INT=

- Rango -> `'1970-01-01 00:00:01'` to `'2038-01-19 03:14:07'`

### YEAR 

- Formato -> `'YYYY'`
- Rango   -> `1901` to `2155`

### TIME

- Formato -> `'HH:MM:SS'`
- Rango   -> `'-828:59:59'` to `'828:59:59'`

### Espacio que ocupa


| Data Type | Before MySql 5.6.4 | After MySql 5.6.7 |
| ---------  | ----------------  | ----------------  |
| YEAR        | 1 byte            | 1 byte            |
| DATE       | 3 bytes            | 3 bytes            |
| TIME       | 3 bytes            | 3 bytes + fractional seconds storage |
| DATETIME   | 8 bytes            | 5 bytes + fractional seconds storage |
| TIMESTAMP  | 4 bytes            | 4 bytes + fractional seconds storage |


#### Fractional Seconds (MySql >= 5.6.4)


| Fractional Seconds Precision | Storage Requider |
|  --------------------------- | ---------------  |
| 0 | 0 bytes |
| 1,2 | 1 bytes |
| 3,4 | 2 bytes |
| 5,6 | 3 bytes |
