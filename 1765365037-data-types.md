# Data Types

## CHAR(n) AND VARCHAR(n)

`CHAR(n)` es un string con una logintud concreta de caracteres dados por `n`.
Si se necesita alguna codificación de algún idioma esta codificación se pone de esta manera => 
`CHARACTER SET ascii` ascii se puede sustituir por cualquier otra codificación com UTF8.

Ejemplos de como se usuaria char =>

```sql
nombre_atributo CHAR(4)

```
Especificando la codificación =>

```mysql
nombre_atributo CHAR(6) CHARACTER SET ascii
```


`VARCHAR(n)` es lo mismo que `CHAR(n)` pero admite tanto números como letras.
es mala pracica usar ==VARCHAR(255)== ya que ocupa mucho espacio en la base de datos y realentiza los select.

## DATE, DATETIME, TIMESTAMP, YEAR and TIME

### DATE

- Formato -> `'YYYY-MM-DD'`
- Rango   -> `'1000-01-01'` to `'9999-12-31'`

### DATETIME

- Formato -> `'YYYY-MM-DD HH:MM:SS'`
- Rango   -> `'1000-01-01 00:00:00'` to `'9999-12-31 23:59:59'`

### TIMESTAMP

==ES DE TIPO INT=

- Rango -> `'1970-01-01 00:00:01'` to `'2038-01-19 03:14:07'`

### YEAR 

- Formato -> `'YYYY'`
- Rango   -> `1901` to `2155`

### TIME

- Formato -> `'HH:MM:SS'`
- Rango   -> `'-828:59:59'` to `'828:59:59'`

### Espacio que ocupa


| Data Type | Before MySql 5.6.4 | After MySql 5.6.7 |
| ---------  | ----------------  | ----------------  |
| YEAR        | 1 byte            | 1 byte            |
| DATE       | 3 bytes            | 3 bytes            |
| TIME       | 3 bytes            | 3 bytes + fractional seconds storage |
| DATETIME   | 8 bytes            | 5 bytes + fractional seconds storage |
| TIMESTAMP  | 4 bytes            | 4 bytes + fractional seconds storage |


#### Fractional Seconds (MySql >= 5.6.4)


| Fractional Seconds Precision | Storage Requider |
|  --------------------------- | ---------------  |
| 0 | 0 bytes |
| 1,2 | 1 bytes |
| 3,4 | 2 bytes |
| 5,6 | 3 bytes |

## Casting Implicito y Automatico

Los casting en MySQL son automaticos y se hacen de la siguiente manera:
 ```sql

  SELECT '123' * 2;

```

El casting va de derecha a izquierda.

En este caso el valor que duevelve el select es 246, como un int. En otras palabras devuelve la 
multiplicación de 123 con 2.

Si la conversión no es posible el resultado es 0.

```mysql

  SELECT 'ABC123' * 2;

```

`Return value: 0`

## Fixed Point Types

### DOUBLE
Se declaran de la sigiente manera: 

```mysql

  atribute DECIMAL(5,2)

```

El 5 son la cantidad total de digitos y el 2 en que digito se pone la coma.
Por ejemplo con la anteriorm declaración solo se puede guardar en la base de datos números en el siguiente
rango: 
  -999.99 al 999.99.

La cantidad maxima de digitos que puede tomar es de 65.

### FLOAT

Es casi igual que el DOUBLE, la unica diferencias significativa es que el FLOAT ocupa 8 bytes y tiene 
una precision de ~7 decimales y DOUBLE ocupa 8 Bytes y tiene una precision de ~16 digitos.

| Type | Storage | Precision |
| ----- | ------ | ------    |
| FLOAT | 4 Bytes | ~7 |
| DOUBLE | 16 Bytes | ~16 |
